## 🛠️ 常见问题排查

### 问题 1: 组件无法识别
**错误信息**: `Component is not found: floating-window`

**解决方案**:
1. 检查 `miniprogram/app.json` 中是否包含:
```json
"usingComponents": {
  "floating-window": "/components/floating-window/floating-window"
}
```
2. 确认组件文件路径正确
3. 重启微信开发者工具

### 问题 2: 浮动窗口不显示
**可能原因**:
- 组件未正确引入到页面
- 页面 .json 文件中缺少组件声明

**解决方案**:
在需要使用浮动窗口的页面 .json 文件中添加:
```json
{
  "usingComponents": {
    "floating-window": "/components/floating-window/floating-window"
  }
}
```

### 问题 3: 匹配度功能无法使用
**错误信息**: `require utils/match.js failed`

**解决方案**:
1. 确认 `utils/match.js` 文件存在
2. 检查 require 路径是否正确：`require('../../utils/match.js')`
3. 确认文件编码为 UTF-8

### 问题 4: 样式异常
**可能原因**:
- app.wxss 未正确加载
- 样式文件路径错误

**解决方案**:
1. 检查 `app.wxss` 是否存在且包含完整设计系统
2. 确认页面 wxss 文件中正确引用了所需样式
3. 清除开发者工具缓存

### 问题 5: 测试结果无法保存
**可能原因**:
- 本地存储权限问题
- 存储空间已满

**解决方案**:
1. 检查微信开发者工具的存储权限设置
2. 清理本地存储：`wx.clearStorageSync()`
3. 使用 try-catch 包裹存储操作

## 🚀 部署步骤

### 步骤 1: 导入项目
1. 打开微信开发者工具
2. 选择「导入项目」
3. 选择项目目录
4. 填写 AppID

### 步骤 2: 编译预览
1. 点击「编译」按钮
2. 检查控制台是否有错误
3. 在模拟器中测试所有功能

### 步骤 3: 功能测试清单
- [ ] 首页正常显示 5 个测试入口
- [ ] 浮动窗口可以打开/关闭
- [ ] 浮动窗口显示正确的测试进度
- [ ] 点击浮动窗口中的测试项可以跳转
- [ ] 所有测试可以正常完成
- [ ] 测试结果可以保存
- [ ] 性格测试完成后可以查看职业匹配度
- [ ] 职业偏好测试完成后可以查看匹配度
- [ ] 结果汇总页显示所有测试数据
- [ ] 分享功能正常

### 步骤 4: 真机预览
1. 点击「预览」按钮

3. 在真机上测试所有功能

1. 点击「上传」按钮
2. 填写版本号和备注
